2. 怎么改

2.1 修改UserService,这个service是一个仓库service，需要将泛型参数User改为自己的用户实体，然后一个个修改查询方法
上面说了，不是说有方法都要改，只需要改上面罗列出来的就行了。

2.2 如果用户不想修改UserService，可以自己实现IUserService接口。然后实现上面几个必须实现的方法。需要说明的是，用户的可以仿照上面写
也可以不仿照上面写。实现类不是必须是仓储类，只需要能把用户信息查到即可。甚至不是必须通过查询数据库来获取用户信息，可以通过http接口来查询
也是可以的。只需要将自己查询出的结果转成BaseIdTranStruVo即可

> 一说到自己实现可能有些用户吓懵了，其实很简单，将自己系统的用户查出以后，转换成BaseIdTranStruVo即可，它只有两个字段id和name，即用户的id和用户的名字
> 工作流引擎只需要知道这些信息已经足够了!
